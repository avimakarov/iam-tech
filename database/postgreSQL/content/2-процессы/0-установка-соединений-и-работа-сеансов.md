При подключении для каждого клиента сервером СУБД создаются отдельные процессы в операционной системе. В этом процессе между клиентом и сервером СУБД устанавливается сеанс - двухсторонняя свзязь, которая позволяет клиенту взаимодействовать с СУБД. После того, как сеанс готов - клиент может отправлять серверу команды
