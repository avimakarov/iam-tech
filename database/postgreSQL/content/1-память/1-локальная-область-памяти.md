Локальные области памяти - сегменты, которые выделяются индивидуально для 
каждого процесса (в рамках сеанса), при этом процессы не имеют доступа к локальным сегментам (областям памяти) других процессов

* Например:
    * **Временные буферы (temp_buffers)**
        * Используются для работы с данными временных таблиц, которые существуют в рамках сеанса или транзакции
    * **Рабочая память процессов (work_mem)**
        * Выделяется при необходимости размещения данных при выполнении промежуточных операций в запросе (сортировка, исключение дубликатов, соединение таблиц по алгоритмам join), если для выполнения операции рабочей памяти недостаточно - на диске создается временный файл, который удаляется после завершения операции
    * **Рабочая память для операций обслуживания (maintenance_work_mem)**
        * Выделяется для таких операций как VACUUM, CREATE_INDEX, REINDEX
